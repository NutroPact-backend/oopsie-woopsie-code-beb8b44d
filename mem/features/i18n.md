---
name: i18n workflow
description: How to make any new UI string translate across all 13 site languages
type: feature
---
**Hook:** `import { useT } from "@/lib/i18n"; const t = useT();`
**Usage:** `{t("cta.add_to_cart")}` — supports `{var}` interpolation via 2nd arg `{ var: value }`.
**Dict file:** `src/lib/i18n.dict.ts` — add new keys to the `en` object. All 13 locale dicts live in same file.
**Fallback:** locale → en → literal key. Safe to ship with only `en` set.
**Regen all locales:** `scripts/gen-i18n-dict.ts` (AI Gateway, gemini-2.5-flash, Amazon.in tone). Keep digits/units/brands in Latin.
**Reuse existing keys** before inventing: `nav.*`, `cta.add_to_cart`, `cta.buy_now`, `cta.checkout`, `footer.*`, `video.views`, `video.watch_now`.
**Admin tabs:** English-only is acceptable; do NOT add `useT` there unless asked.
**Heading/subheading from DB** (e.g. video_sections.heading): translate by storing per-locale variants in the admin record OR pass through `getProductTranslation`-style server fn. Do not hardcode.
