# Three layers — use the right one

## 1. Short string (developer-written OR admin DB short field) → `<T>` or `useAutoT`
Easiest, zero setup. Auto-translates on first render, caches forever in `content_translations` table (entityType `ui_string`). Cap **1000 chars** (covers FAQ answers, blog excerpts, banner copy). Longer strings: see layer 3.
```tsx
import { T, useAutoT } from "@/lib/useContentT";
<T>Add to Cart</T>
<T>{product.name}</T>           // admin DB short field is fine here too
const label = useAutoT("Continue shopping");
```
Use this for EVERY new visible English string AND for short admin DB fields (titles, names, headings, labels, FAQ Q/A).

## 2. Curated key with reviewed copy → `useT(key)` + i18n.dict.ts
For high-traffic strings where you want hand-reviewed translations.
```tsx
import { useT } from "@/lib/i18n";
const t = useT(); {t("cta.add_to_cart")}
```
Add key to `en` in `src/lib/i18n.dict.ts`; regen with `scripts/gen-i18n-dict.ts`.
Reuse existing: `cta.add_to_cart`, `cta.buy_now`, `cta.checkout`, `nav.*`, `footer.*`, `video.views`.

## 3. LONG admin DB content (page bodies, blog post body, custom HTML) → `useContentT` / `useContentTBatch`
For long-form admin content (>1000 chars) OR when you want a stable cache row keyed by entity (so editing source invalidates cleanly). Auto-translates first viewer, caches per (entity, field, locale).
```tsx
import { useContentT, useContentTBatch } from "@/lib/useContentT";
// Single:
const body = useContentT("page_block", "privacy", "content", page.content);
// Batch (list rendering):
const ct = useContentTBatch([
  { entityType: "video_section", entityId: s.id, field: "heading", source: s.heading },
]);
ct("video_section", s.id, "heading");
```

## Stable entityType convention (use these — don't invent new ones lightly)
- `product` (name, description, short_desc)
- `blog_post` (title, excerpt, content)
- `faq_item` (question, answer)
- `page_block` (title, content) — privacy, terms, refund, custom pages
- `home_section` (heading, subheading, cta_label)
- `banner` (text)
- `category` (name, description)
- `nav_item` (label)
- `testimonial` (text)
- `combo` (name, description)
- `video_section` (heading, subheading)

**Rule for every new admin-built section/page**: short fields → `<T>{x}</T>`, long body → `useContentT(entityType, id, field, source)`.

# Infrastructure
- Table: `public.content_translations` (entity_type, entity_id, field, locale, source_hash, translated)
- Server fn: `getContentTranslations` in `src/lib/content-translations.functions.ts` → delegates to `translateStrings()` in `src/lib/translation-provider.server.ts`
- **Provider priority**: `GOOGLE_TRANSLATE_API_KEY` (Google Cloud Translation v2 — 92-95% accuracy) → AI gateway fallback via `getAIConfig()` (Gemini 2.5 Flash — 80-85% accuracy). To switch engines just add/remove the secret.
- Fallback chain: provider → cache → source string (UI never blocks)
- Cache invalidates automatically when source string changes (source_hash mismatch → re-translate on next view)
- Client batches >50 items are auto-chunked (50/call, parallel) — `useContentTBatch` handles this internally.
